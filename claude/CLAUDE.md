# Claude Code 指示書

全プロジェクト共通の開発ガイドラインと好みを定義します。

## コミュニケーションスタイル

- 日本語で応答する
- 簡潔で直接的に
- 明示的にリクエストされない限り絵文字は使わない
- 技術的正確性を重視する
- 前提と矛盾していても客観的なガイダンスを提供する

## コードスタイル・規約

### 基本原則
- 賢さより明確さを優先する
- 意味のある変数名・関数名で自己文書化コードを書く
- 関数は小さく、単一責任に集中させる
- 深いネストを避け、早期リターンを使う

### コメント・ドキュメント
- 「なぜ」を書き、「何を」は書かない
- 複雑なアルゴリズムやビジネスロジックは文書化する
- コード変更時にコメントも更新する

### テスト
- 新機能とバグ修正にはテストを書く
- テストは読みやすく保守しやすくする
- シナリオを説明する記述的なテスト名を使う

## Git プラクティス

### コミットメッセージ
- Conventional Commit フォーマットを使用: `type: description`
- タイプ: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`
- 1行目は50文字以内
- 必要に応じて本文で詳細を説明

## 一般的な注意事項

- 明示的な要求がない限り設定ファイルは変更しない
- "DO NOT MODIFY" コメントがあるコードには触れない
- 本番環境の環境変数やシークレット情報は慎重に扱う
- セキュリティの脆弱性（SQLインジェクション、XSS等）を導入しない
