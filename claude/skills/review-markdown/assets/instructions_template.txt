### API Documentation Review

```
Please review this API documentation with focus on:

1. **Completeness** - All endpoints documented?
2. **Clarity** - Request/response formats clearly explained?
3. **Examples** - Are code examples provided for each endpoint?
4. **Error handling** - Are error codes and messages documented?
5. **Authentication** - Is auth mechanism clearly explained?

Use severity levels:
- must: Critical API issues
- should: Important documentation gaps
- suggestion: Improvements for usability
- question: Clarifications needed
```

### Technical Design Document

```
Please evaluate this design document on:

1. **Architecture clarity** - Is the system design clearly explained?
2. **Component relationships** - Are dependencies and interactions clear?
3. **Decision rationale** - Are design decisions justified?
4. **Trade-offs** - Are trade-offs discussed?
5. **Implementation feasibility** - Is the design implementable?

Severity:
- must: Unclear or impractical design
- should: Missing important considerations
- suggestion: Potential improvements
- question: Points needing clarification
```

### README Documentation

```
Please check this README for:

1. **Project overview** - Clear description of what the project does?
2. **Getting started** - Is setup straightforward?
3. **Usage examples** - Are usage examples clear and correct?
4. **Requirements** - Are dependencies and prerequisites listed?
5. **Contributing** - Are contribution guidelines clear?

Severity:
- must: Critical information missing
- should: Important sections unclear
- suggestion: Enhancement opportunities
- question: Unclear content
```

### Generated Content Quality Check

```
Please verify this AI-generated content for:

1. **Factual accuracy** - No hallucinations or false statements?
2. **Coherence** - Does content flow logically?
3. **Tone consistency** - Is tone appropriate and consistent?
4. **Completeness** - Are all key points covered?
5. **Usability** - Is content practical and actionable?

Severity:
- must: Factual errors that must be corrected
- should: Content improvements needed
- suggestion: Enhancement opportunities
- question: Verify with domain expert
```
