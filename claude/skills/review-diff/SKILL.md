  以下の場面で使用：
  - ユーザーが「差分をレビューして」「diff をレビュー」「コードレビューして」と依頼した場合
  - ユーザーが「変更点をチェックして」「commit 前にレビュー」と依頼した場合
  - コミット直後に「レビューしてほしい」と依頼した場合
  - "/review-diff" を手動で実行した場合
  - "review diff", "レビュー", "review changes" のキーワードを含む依頼

  Workflow:
  1. プロジェクトのドキュメント・ルールを事前に読み込み、プロジェクトの文脈を把握
  2. git diff で現在のブランチの差分を取得
  3. 7 つの専門エージェントを並列起動してレビュー実施
     - Security: セキュリティ脆弱性・認証・認可
     - Performance: パフォーマンス・計算量・メモリ
     - Code Quality: 可読性・設計・命名・重複
     - Architecture: 設計パターン・依存関係・モジュール性
     - Testing: テストカバレッジ・テスト品質・エッジケース
     - Documentation: コメント・docstring・ログ・可観測性
     - Error Handling: 例外処理・タイムアウト・堅牢性
  4. 各エージェントの結果を集約して総合レポートを生成
  5. 優先度付きの改善提案を提示

  IMPORTANT: コミット後にこのスキルを提案するため、
  コミット直後に「このコミットをレビューしますか？」と自動提案することも検討する。

# コード差分レビュースキル

**IMPORTANT**: このスキルは Git 差分の高品質なコードレビューを 7 並列エージェントで実施します。

---

## Phase 0: 事前準備 — プロジェクト文脈の把握

**レビュー開始前に必ず以下を実行してください。**

### ステップ 0-1: プロジェクトルール・ドキュメントの読み込み

以下のファイルを順番に確認し、プロジェクトの文脈を完全に理解してください：

```bash
# CLAUDE.md（プロジェクト固有ルール）が存在するか確認
ls CLAUDE.md 2>/dev/null || ls .claude/CLAUDE.md 2>/dev/null

# README.md を確認
ls README.md 2>/dev/null

# package.json, pyproject.toml, go.mod などのプロジェクト設定
ls package.json pyproject.toml go.mod Cargo.toml pom.xml 2>/dev/null
```

読み込むべきファイル（存在するものを優先度順で読む）：
1. `CLAUDE.md` または `.claude/CLAUDE.md` — **必読**（プロジェクト固有ルール）
2. `README.md` — プロジェクト概要・技術スタック
3. プロジェクト設定ファイル（package.json, go.mod 等）— 依存関係・バージョン

### ステップ 0-2: コーディングガイドライン確認

```bash
# スタイルガイド・コーディング規約を確認
find . -name ".editorconfig" -o -name ".eslintrc*" -o -name "*.prettierrc*" \
       -o -name "pylintrc" -o -name ".flake8" 2>/dev/null | head -5
```

## Phase 1: 差分の取得

### ステップ 1-1: 現在の状況把握

```bash
# 現在のブランチ確認
git branch --show-current

# コミット履歴確認（直近 10 件）
git log --oneline -10

# ステージング済み・未ステージの変更確認
git status
```

### ステップ 1-2: 差分の取得

引数があれば指定ブランチとの差分、なければデフォルト動作：

```bash
# 引数なし: ベースブランチ（main/master）との差分
BASE_BRANCH=$(git remote show origin 2>/dev/null | grep 'HEAD branch' | awk '{print $NF}' || echo "main")

# ステージング済み + 未ステージ済みの全差分
git diff HEAD

# または main/master との差分（ブランチ作業時）
git diff ${BASE_BRANCH}...HEAD

# 変更ファイル一覧
git diff --name-only HEAD
```

**$ARGUMENTS が指定された場合**: そのコミットハッシュまたはブランチとの差分を取得

## Phase 2: 7 並列レビュー実行

差分を取得したら、以下の 7 つの観点で **並列に** Task ツールを使ってサブエージェントを起動してください。

**重要**: 7 つのエージェントを同時に起動することで高速・包括的なレビューを実現します。

### 各エージェントへの指示テンプレート

以下の 7 つのエージェントを **一度に** 並列起動してください：

---

## コミット後の自動提案

**コミット操作（Bash での git commit）を検出した場合**、レビューを提案してください：

```

このコミットの差分をレビューしますか？

```

## 使用方法

### 手動実行
```
/review-diff                    # 現在の差分（HEAD vs ベースブランチ）
/review-diff HEAD~1             # 直前のコミットの差分
/review-diff main               # main ブランチとの差分
/review-diff abc123             # 特定コミットとの差分
```

### Codex MCP 経由での実行

このスキルは Codex MCP ツールを通じて実行することもできます。
`mcp__codex__codex` ツールを使ってサブエージェントとして実行：

```
mcp__codex__codex を使って以下を実行:

sandbox: "read-only"
```

## トラブルシューティング

### 差分が大きすぎる場合

```bash
# 変更ファイルのみ確認
git diff --name-only HEAD

# 特定ファイルのみレビュー
git diff HEAD -- path/to/file.ts
```

### ベースブランチが不明な場合

```bash
# リモートのデフォルトブランチを確認
git remote show origin | grep 'HEAD branch'

# 全ブランチを確認
git branch -a
```

### Codex MCP が利用できない場合

通常の Task ツールでサブエージェントを起動してください。
同じ並列レビューを `subagent_type: general-purpose` で実施可能です。
