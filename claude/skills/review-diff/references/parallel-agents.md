# 5 並列エージェント実行ガイド

このドキュメントは `review-diff` スキルで 5 つのエージェントを並列実行する方法を説明します。

---

## 並列実行の原則

Claude Code では、**1 つのメッセージで複数の Task ツールを呼び出す**ことで並列実行が可能です。

### 基本パターン

```
# 1 つのメッセージで 5 つの Task を同時起動
Task(security_review)     ─┐
Task(performance_review)  ─┤
Task(quality_review)      ─┼─ 並列実行
Task(architecture_review) ─┤
Task(testing_review)      ─┘
          ↓
    結果を集約してレポート生成
```

---

## 実装時の注意点

### 差分データの共有

5 つのエージェントは同じ差分データをそれぞれ受け取ります。
差分が大きい場合は、事前にファイルに保存してパスを渡す方が効率的です：

```bash
# 差分をファイルに保存
git diff HEAD > /tmp/current.diff

# 各エージェントに diff のパスを渡す
```

### エージェントタイプの選択

| エージェント | 推奨タイプ | 理由 |
|------------|----------|------|
| Security | `security` | セキュリティ特化ツール |
| Performance | `performance` | パフォーマンス特化ツール |
| Code Quality | `reviewer` | コードレビュー特化ツール |
| Architecture | `architect` | アーキテクチャ設計特化 |
| Testing | `qa` | テスト品質特化ツール |
| Documentation | `reviewer` | ドキュメント品質レビュー |
| Error Handling | `analyzer` | 根本原因分析・堅牢性評価 |

### フォールバック

専用エージェントが利用できない場合は `general-purpose` を使用してください。

---

## Codex MCP との連携

`mcp__codex__codex` ツールを使用してサブエージェントを起動する場合：

```
mcp__codex__codex(
  prompt: "以下の diff を[観点]でレビューしてください:\n[diff]",
  sandbox: "read-only",
  approval-policy: "never"
)
```

### Codex MCP の利点
- 完全に独立したサンドボックス環境での実行
- メインコンテキストを消費しない
- 読み取り専用モードでの安全なレビュー

### Codex MCP のスレッド管理

```
# 最初の呼び出し
result = mcp__codex__codex(prompt: "...", sandbox: "read-only")
threadId = result.threadId

# 後続の呼び出し（同じスレッドで継続）
mcp__codex__codex-reply(threadId: threadId, prompt: "詳細を教えてください")
```

---

## エラーハンドリング

### 一部のエージェントが失敗した場合

1. 失敗したエージェントの結果を「取得できませんでした」と記録
2. 他の 4 つのエージェントの結果で部分的なレポートを生成
3. ユーザーに失敗したカテゴリを明示して再実行を提案

### タイムアウト

各エージェントは最大 10 分で完了する想定。
タイムアウトした場合はそのエージェントをスキップして集約。

---

## 結果集約のタイミング

5 つのエージェントが全て完了（または失敗）した後、
メインエージェントが結果を集約してレポートを生成します。

集約時の優先順位：
1. **Critical 項目のマージ**: 重複を排除して優先度順にソート
2. **Warning 項目のマージ**: カテゴリ別にグループ化
3. **Info 項目のマージ**: 情報密度を調整
4. **良かった点の統合**: 全エージェントの正のフィードバック

---

## 差分サイズの制限

| 差分サイズ | 対応 |
|-----------|------|
| ~200行 | 全差分を各エージェントに渡す |
| 200~500行 | 差分をファイル単位で分割して渡す |
| 500行以上 | ユーザーにファイル指定を依頼 |

大きな差分の場合は以下のようにユーザーに提案：

```
差分が [X] 行と大きいです。
以下のオプションを選択してください：
1. 全差分を一括レビュー（時間がかかる場合があります）
2. 変更ファイルを選択してレビュー: [ファイル一覧]
3. 最重要ファイルのみレビュー
```
